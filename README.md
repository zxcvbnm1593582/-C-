C 语言贪吃蛇游戏优化版：从开源项目出发
项目名称
C语言贪吃蛇游戏优化版
一、项目介绍。
这是一个根据作者"ewoifuoi"所制作的贪吃蛇游戏稍作修改而生的作品，增添了作品的灵活性和趣味性。
二、环境配置（含工具开发基础）
1、操作系统：Windows（Win7/Win10/Win11）
2、编译器：支持 Windows API 的 C 编译器：
. Visual Studio 2019/2022
. MinGW-w64（需配置环境变量）
. Dev-C++（5.11 及以上版本）
3、字符编码：GBK/GB2312（Windows 控制台默认编码，避免中文乱码）
4、控制台权限：无需管理员权限，普通用户即可运行
三、运行步骤
在VS2022中打开此代码点击调试，之后按系统题诗操作即可
四、功能说明
可用于短时间的休闲娱乐
五、个人修改/扩展（含工具、算法、数据处理）
1、皮肤添加部分
  首先代码中通过enum定义了皮肤类型枚举SkinType，把不同皮肤抽象成具有语义的常量，替代魔法数字，让代码更易读、易维护；其次，定义了两个全局数组，建立从枚举值到显示字符，从枚举值到皮肤名称的一一对应关系；然后showSkinMenu() 函数实现皮肤选择界面；接着定义 getSkinChar() 函数封装皮肤字符的获取逻辑，所有绘制蛇身或蛇头的地方，都调用这个函数获取当前选中的皮肤字符；最后，进入皮肤选择菜单，用户通过↑↓选择、回车确认，把选中的皮肤枚举值赋值给s.skin实现初始化，游戏中所有绘制蛇头或蛇身的场景，都通过 getSkinChar(s.skin) 获取当前皮肤字符，只要 s.skin 的值不变，蛇的显示字符就固定，重新选择皮肤时，s.skin 被重新赋值，后续绘制会立即使用新字符。
2、蛇身移动逻辑部分
    mark 数组优化：仅更新移动时变化的蛇头或蛇尾位置，替代每帧全量遍历赋值，时间复杂度从 O (n)→O (1)，蛇越长性能提升越明显
3、数据处理，优化为动态内存分配
   将蛇身的固定数组 node[101] 替换为动态内存分配的指针数组，通过 malloc/calloc/realloc 实现蛇身节点的动态扩容，解除最大长度限制；同时补充内存管理逻辑，避免内存泄漏。
